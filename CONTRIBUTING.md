# Contributing to StreamPilot

Thank you for helping build StreamPilot.

Before contributing, please read:

1. `PROJECT.md`
2. `docs/SPS/SPS-000-Manifesto.md`
3. `docs/SPS/SPS-014-Engineering-Principles.md`

## Contribution Principles

Contributions should:

- Help creators
- Respect privacy
- Keep AI optional
- Preserve local ownership
- Improve reliability
- Include documentation when behavior changes

## Workflow

1. Open or comment on an issue.
2. Discuss large changes before implementation.
3. Create a feature branch.
4. Add tests when practical.
5. Update documentation.
6. Open a pull request.

## Definition of Done

A feature is done when:

- Code is implemented.
- Tests pass.
- Documentation is updated.
- Security implications are considered.
- Changelog is updated if user-facing.
- The change supports StreamPilot's mission.
