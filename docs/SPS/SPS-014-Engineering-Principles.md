# SPS-014 — Engineering Principles

**Status:** Approved  
**Version:** 1.0

## Principles

- Creator First
- Simplicity Over Cleverness
- Local First
- AI Is Optional
- Security by Design
- Small, Composable Components
- Clear Boundaries
- Explicit Over Implicit
- Fail Gracefully
- Documentation Is Part of the Product
- Backward Compatibility Matters
- Testability
- Community Friendly
- Continuous Improvement
- Respect Creator Focus
- Build for Longevity
- Build With Purpose

## Decision Framework

Ask:

1. Does this benefit creators?
2. Does it align with the Manifesto?
3. Is it the simplest reliable solution?
4. Does it preserve security?
5. Does it respect creator ownership?
6. Is it maintainable?
7. Would a new contributor understand it?
8. Will this still make sense in five years?

## Guiding Principle

Build software that creators can trust, contributors can understand, and the community can proudly grow together.
